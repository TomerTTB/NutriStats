Security Overview

Food Diary takes security seriously. This section outlines the comprehensive security measures implemented to protect your personal data and ensure safe usage of the application.

Security Architecture

Multi-Layer Security Approach

Our security implementation follows industry best practices with multiple layers of protection:

Security Layers

  • Authentication Layer - JWT-based stateless authentication
  • Authorization Layer - Role-based access control
  • Data Layer - Encrypted passwords and secure data storage
  • Transport Layer - HTTPS encryption (production)
  • Application Layer - Input validation and sanitization

Key Security Features

üîê JWT Authentication

We use JSON Web Tokens (JWT) for secure, stateless authentication:

  • Stateless Design - No server-side session storage required
  • Token Expiration - Automatic timeout for security
  • Refresh Tokens - Secure token renewal mechanism
  • Database Verification - Tokens validated against user database

üîí Password Security

Your passwords are protected using industry-standard encryption:

  • Bcrypt Hashing - One-way encryption with salt
  • 12 Salt Rounds - High computational cost for attackers
  • Password Strength Requirements - Enforced complexity rules
  • No Plain Text Storage - Passwords never stored in readable form

Multiple mechanisms protect against common security threats:

  • Rate Limiting - Prevents brute force login attempts
  • SQL Injection Protection - Parameterized database queries
  • Input Validation - All user input sanitized and validated
  • XSS Prevention - Output encoding and content security

Data Protection

Personal Information

Your personal data is protected through:

  • User Isolation - Each user's data is completely separate
  • Minimal Data Collection - Only necessary information is stored
  • Secure Storage - SQLite database with proper permissions
  • No Third-Party Sharing - Your data stays with you

Nutrition and Weight Data

Your health information receives special protection:

  • User-Specific Access - Only you can see your data
  • Encrypted Transmission - HTTPS in production
  • Local Storage - Data stored on your chosen server
  • Export Capability - You own and control your data

Authentication Flow

Registration Process

  1. User provides email, name, and password
  2. Password strength is validated
  3. Email uniqueness is verified
  4. Password is hashed with bcrypt
  5. User account is created in database
  6. JWT tokens are generated and returned

Login Process

  1. User provides email and password
  2. Rate limiting check is performed
  3. User is retrieved from database
  4. Password is verified against hash
  5. JWT tokens are generated
  6. Failed attempts are tracked

API Request Protection

  1. JWT token is extracted from Authorization header
  2. Token signature and expiration are verified
  3. User existence is confirmed in database
  4. Request is authorized for the specific user
  5. User context is added to request

Security Configuration

Environment Variables

Critical security settings are configured through environment variables:

# JWT Configuration
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
JWT_EXPIRES_IN=24h
JWT_REFRESH_SECRET=your-refresh-secret-key-different-from-main
JWT_REFRESH_EXPIRES_IN=7d

# Security Settings
BCRYPT_SALT_ROUNDS=12
MAX_LOGIN_ATTEMPTS=5
LOCKOUT_TIME_MINUTES=15

# Database Security
DB_PATH=./src/data/nutrition_app.db
NODE_ENV=production

Security Monitoring

What We Log

  • Failed login attempts with IP addresses
  • Successful authentication events
  • API access patterns and errors
  • Database connection issues
  • Security-related errors and warnings

What We Don't Log

  • Passwords (plain text or hashed)
  • JWT token contents
  • Personal nutrition data details
  • Sensitive user information

Incident Response

If You Suspect a Security Issue

  1. Change Your Password - Immediately update your password
  2. Log Out All Sessions - Use the logout functionality
  3. Check Your Data - Review your recent activity
  4. Report the Issue - Contact the system administrator

Common Security Indicators

  • Unexpected login notifications
  • Changes to your data you didn't make
  • Unusual system behavior
  • Requests for sensitive information

Privacy Commitment

Data Ownership

You maintain complete ownership of your data:

  • Your nutrition and weight data belongs to you
  • You can export your data at any time
  • You can delete your account and all associated data
  • No third-party data sharing without explicit consent

Transparency

We believe in security through transparency:

  • Open source security implementations
  • Clear documentation of security measures
  • Regular security updates and improvements
  • Community-driven security enhancements
